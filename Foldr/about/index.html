About Foldr

A tidy desktop, on autopilot. A quiet, local-first organizer for people who would rather not hand their file names to somebody's server.

The Story

Why this exists.

Origin Story

This project started with a folder that had quietly turned into a landfill — installers from two years ago, screenshots nobody named, and the one invoice that always went missing exactly when it was needed.

Everything on the market was either heavy, rented by the month, or eager to upload file names somewhere. The wish list was short: something that works in the background, never phones home, and never asks for attention.

Project Goals

This is not an attempt to build the most powerful file manager on earth. The target is the calmest and most boring way to keep one folder in order.

  • Useful the second it launches, no setup
  • Runs entirely offline
  • No accounts, no subscriptions, no ads
  • Never steals focus with pop-ups

Philosophy

Local and private. Nothing is transmitted off the machine. Rules, history, and settings live in a local SQLite file. System entries such as desktop.ini, Thumbs.db, and .DS_Store are left untouched.

Open source (MIT). The source is public — read it, fork it, ship your own build. There is nothing in there worth hiding.

No interruptions. It waits in the tray and speaks only through a small toast once something has been filed.

Under the Hood

The stack is deliberately unexciting and dependable:

  • Rust core — folder watcher, SQLite, rule engine
  • Tauri 2.x — small native shell around it
  • React 19 + TypeScript — quick, type-safe interface
  • Tailwind CSS — tidy, low-maintenance styling

The whole thing lands around 3 MB and idles on a rounding error of RAM.

Getting Started

Windows: grab the auto-installer (.exe) or the MSI from the homepage and run it. The app launches on its own and drops an icon into the system tray.

Linux: grab the AppImage, .deb, or .rpm from the homepage, then follow the Linux notes below.

1. Start the app. Out of the box it watches your Downloads folder and nothing else.

2. Left-click the tray icon for the popup — recent moves, weekly stats, and a manual clean button.

3. Right-click the tray icon for the menu: Clean Now, Settings, Quit.

4. Drop any file into the folder and watch it land in the correct subfolder a couple of seconds later.

Linux Installation

Most current distributions work out of the box. Two libraries need to be present — on a normal desktop install they already are:

  • libwebkit2gtk-4.1
  • libayatana-appindicator3

AppImage (universal, nothing to install):

chmod +x Foldr_0.1.0_amd64.AppImage
./Foldr_0.1.0_amd64.AppImage

.deb (Debian, Ubuntu, Mint, Pop!_OS):

sudo dpkg -i Foldr_0.1.0_amd64.deb
sudo apt-get install -f

.rpm (Fedora, openSUSE, RHEL):

sudo dnf install Foldr-0.1.0-1.x86_64.rpm

On distros with AppArmor or comparable sandboxing, grant access to your home directory so folders can be watched and files moved.

Features

Quiet by default — background operation with no pop-ups; a small toast is the only signal that anything happened.

Rule engine — build rules from extensions and regex patterns, and route to dynamic destinations such as {year}/{month}.

History & undo — every move is written to a local log. Single-click undo, or Revert All from the History tab.

7 languages — the interface follows your system locale automatically.

Dark mode — tracks the OS theme, or pin Light / Dark / System yourself.

Autostart — registers itself at login; switch it off under Settings → General.

Manual and paused folders — let a folder collect files without moving them, or suspend watching altogether.

Scheduled cleaning — trigger Clean Now automatically at up to four fixed times a day.

Adjustable grace period — hold moves anywhere from a few seconds to 48+ hours.

Rule export / import — rules travel as plain JSON.

Configuration

Open Settings from the tray menu to adjust everything below.

Watched folders — add or drop folders from the watch list. Only Downloads is watched initially.

Rules — match on extensions, regex, or wildcards, then build the destination path from placeholders:

  • {year} — the current year (e.g. 2026)
  • {month} — the current month (e.g. 07)
  • {filename} — the original name, extension stripped

General — language, theme, and autostart behavior.

Archive Import

Large data-export archives can be handed straight to the app instead of being unpacked and sorted by hand.

  1. Download your archive (typically .zip, .tgz, or .tar.gz).
  2. Open Settings → Folders.
  3. Choose Import archive and pick the file.
  4. The archive is unpacked into a temporary staging folder, and your existing rules run across the contents.
  5. Files move to their destinations. Staging is kept afterwards so you can review it and delete it yourself.

The archive itself can sit on a network or external drive — only the extraction needs local space. Big archives may take a minute or two.

Ignore Rules

Specific files and folders can be skipped through ignore rules, set either in Settings → Ignore or by hand in an .appignore file inside any watched folder.

How it works. Before touching a new file, the app reads .appignore in that folder. A name matching any rule is left exactly where it is.

Adding rules in the app. Settings → Ignore, pick a watched folder, type the pattern, press Enter or Add, then Save. The .appignore file is created or updated for you.

Writing rules by hand. Make a plain text file called .appignore in the folder. One rule per line; lines beginning with # are comments.

Rule types:

  • Exact name — desktop.ini skips that one file.
  • Wildcard — *.tmp skips every .tmp; ~$* skips Office temp files.
  • Folder — node_modules/ (trailing slash) skips the whole folder.

Typical file:

# Office temp files
~$*

# every .tmp
*.tmp

# whole folder
node_modules/

# macOS clutter
.DS_Store

# Windows thumbnails
Thumbs.db

Notes:

  • Matching is case-insensitive on Windows.
  • Every watched folder can carry its own .appignore.
  • Delete the file and everything in that folder becomes sortable again.
  • Common system files are skipped by default — no need to list them.

Privacy & Security

The app is local-first, which in practice means:

  • No cloud connections
  • No telemetry, no analytics
  • Everything in a local SQLite database
  • No account, no registration

Independent scans — publish your build's scan report here so anyone can verify it. Detections on a brand-new unsigned binary are usually heuristic false positives.

SmartScreen — Windows warns about unsigned installers. "More info" → "Run anyway" gets past it.

FAQ

How does it work?
It watches the folders you choose from the tray. A new file gets a two-second pause to confirm the transfer finished, then moves according to your rules.

Does any data leave my machine?
No. Everything runs offline.

Can I customize the rules?
Yes — Settings → Rules for your own sorting logic.

What if a file lands in the wrong place?
Settings → History, then undo. One click.

Does it start with Windows?
Yes, and it can be turned off in Settings → General.

Is it really free?
Yes. MIT License — use it, change it, build your own.

Troubleshooting

Nothing is being sorted.
Check that the folder is in Watched Folders and that some rule actually matches the extension.

Undo is greyed out.
The moved file has to still exist. Delete or move it by hand and there is nothing left to restore.

SmartScreen blocks the installer.
Expected for unsigned apps: "More info" → "Run anyway".

CPU or RAM spikes.
Look at your watched folders — network drives and folders holding thousands of files are the usual culprits.
