Ember: установка и работа с библиотекой

Обновлено: июль 2026 · Репозиторий · API-справочник

Что это

Ember — Python-библиотека и консольная утилита для извлечения и скачивания медиа из соцсетей. Даёшь ссылку на пост — получаешь прямые ссылки на файлы с метаданными; скачать Ember умеет и сам, включая HLS-потоки, без yt-dlp. Поддерживает 15 сервисов: TikTok, Twitter/X, Instagram, Reddit, Vimeo, SoundCloud, Pinterest, Tumblr, Bluesky, Newgrounds, Rutube, OK.ru, VK, Facebook и клипы Twitch.

Шаг 1. Установка

  1. Нужен Python 3.9+.
  2. Пакета на PyPI нет — ставь из Git:
pip install git+https://github.com/smeshidojoe/Ember.git

После установки доступны и Python API (import ember), и команда ember в терминале. Единственная обязательная зависимость — requests.

ffmpeg нужен не всегда: прямые mp4 и простые HLS качаются без него. Он требуется для склейки раздельных видео+аудио дорожек, режима «только звук» и вшивания метаданных. Достаточно положить ffmpeg в PATH.

Шаг 2. Первые команды (CLI)

Главное правило: без флага -d ничего не скачивается — команда только печатает прямые ссылки и метаданные. Ссылку бери в кавычки.

# посмотреть ссылки и метаданные
ember "https://x.com/user/status/123456789"

# то же самое в JSON
ember "https://www.tiktok.com/@user/video/7123456789" --json

# СКАЧАТЬ: -d включает загрузку
ember -d "https://vimeo.com/76979871"

# своё имя файла (-o) и папка (-p)
ember -d -o myclip -p downloads "https://vimeo.com/76979871"

Шаг 3. Качество, звук, плейлисты

# ограничить качество и качать HLS в 6 потоков
ember -d --max-height 720 --concurrency 6 "https://rutube.ru/video//"

# только звук + метаданные в файл (нужен ffmpeg)
ember -d --audio-only --embed-metadata "https://soundcloud.com/user/track"

# список доступных качеств без скачивания
ember -F "https://vimeo.com/76979871"

# целый плейлист (пока SoundCloud-сеты)
ember -d --playlist "https://soundcloud.com/user/sets/name"

# последние посты автора по ссылке на профиль
ember -d --timeline --limit 30 "https://x.com/nasa"

Пачка ссылок из файла: ember -d -a links.txt (одна ссылка на строку, # — комментарий). Полный список флагов — ember -h.

Шаг 4. Куки: приватный и NSFW-контент

Часть контента отдаётся только залогиненным: NSFW-твиты, приватные и полнокачественные посты Instagram, публичные видео Facebook. Ember умеет брать куки прямо из браузера:

ember -d --cookies-from-browser firefox "https://x.com/user/status/123"
  • Рекомендуется Firefox — куки читаются из коробки на любой ОС.
  • Из не-хромовских Chromium-браузеров на Windows тоже работает: Vivaldi, Opera.
  • Chrome/Edge/Brave последних версий шифруют куки (App-Bound Encryption) — чтение из них чаще всего не работает. Не мучайся, поставь Firefox.
Совет: отдельный браузер под куки. Поставь браузер, которым не пользуешься (например Firefox или Vivaldi), войди в аккаунты сервисов, с которых собираешься качать, — и больше его не открывай. Куки останутся валидными, основной браузер не при делах, а залогиненная сессия не будет случайно разлогинена.

Альтернативы: куки строкой --cookies "auth_token=...; ct0=..." или файлом --cookies-file cookies.txt (формат Netscape, как у yt-dlp). Профиль браузера выбирается через --browser-profile.

Шаг 5. Python API

Базовый цикл — извлечь и скачать:

import ember

result = ember.extract("https://vimeo.com/76979871")
print(result.title, result.author)
for m in result.media:
    print(m.kind, m.quality, m.url)

paths = ember.download(result, "downloads/",
                       max_height=720, concurrency=6,
                       embed_metadata=True)

Полезные проверки:

ember.can_extract(url)                      # поддерживается ли ссылка
ember.available_qualities(result.media[0])  # например [1080, 720, 480]
ember.ffmpeg_available()                    # есть ли ffmpeg
ember.probe_size(result.media[0])           # размер файла до скачивания

Куки в API: ember.extract(url, cookies_from_browser="firefox") или словарём cookies={"auth_token": "..."}. Прогресс — колбэком on_progress, плейлисты — extract_playlist(), посты автора — extract_timeline(url, limit=30).

Типовой паттерн встраивания — Ember первым, yt-dlp запасным:

if not ember.can_extract(url):
    return run_ytdlp(url)
try:
    result = ember.extract(url)
except ember.EmberError:
    return run_ytdlp(url)
ember.download(result, out_dir)

Все ошибки наследуются от ember.EmberError — лови его одного. Подробности по каждой функции и моделям данных (Result, Media, Playlist) — в API.md.

Ограничения

  • Instagram — анонимно часто отдаёт только превью; с куками — полное качество.
  • Reddit, Newgrounds, OK.ru — блокируют запросы с датацентровых/VPN-адресов; с домашнего IP работают. Поможет --proxy.
  • Twitter/X — NSFW-твиты требуют куки аккаунта.
  • YouTube — вне поддержки, для него бери yt-dlp.

Ember: installation and usage

Updated: July 2026 · Repository · API reference

What it is

Ember is a Python library and CLI for extracting and downloading media from social platforms. Give it a post URL — it returns direct media links with metadata, and can download by itself too, including HLS streams, without yt-dlp. Supports 15 services: TikTok, Twitter/X, Instagram, Reddit, Vimeo, SoundCloud, Pinterest, Tumblr, Bluesky, Newgrounds, Rutube, OK.ru, VK, Facebook and Twitch clips.

Step 1. Installation

  1. Python 3.9+ required.
  2. Not on PyPI — install from Git:
pip install git+https://github.com/smeshidojoe/Ember.git

Both the Python API (import ember) and the ember command become available. The only required dependency is requests.

ffmpeg is not always needed: direct mp4 and plain HLS download without it. It's required for muxing separate video+audio tracks, audio-only mode and embedding metadata. Just put ffmpeg on PATH.

Step 2. First commands (CLI)

Key rule: without -d nothing downloads — the command only prints direct links and metadata. Quote the URL.

# show direct links and metadata
ember "https://x.com/user/status/123456789"

# same as JSON
ember "https://www.tiktok.com/@user/video/7123456789" --json

# DOWNLOAD: -d enables it
ember -d "https://vimeo.com/76979871"

# custom file name (-o) and folder (-p)
ember -d -o myclip -p downloads "https://vimeo.com/76979871"

Step 3. Quality, audio, playlists

# cap quality, fetch HLS in 6 threads
ember -d --max-height 720 --concurrency 6 "https://rutube.ru/video//"

# audio only + embed metadata (needs ffmpeg)
ember -d --audio-only --embed-metadata "https://soundcloud.com/user/track"

# list available qualities without downloading
ember -F "https://vimeo.com/76979871"

# a whole playlist (SoundCloud sets for now)
ember -d --playlist "https://soundcloud.com/user/sets/name"

# an author's latest posts by profile URL
ember -d --timeline --limit 30 "https://x.com/nasa"

Batch from a file: ember -d -a links.txt (one URL per line, # comments). Full flag list — ember -h.

Step 4. Cookies: private and NSFW content

Some content is only served to logged-in users: NSFW tweets, private and full-quality Instagram posts, public Facebook videos. Ember can read cookies straight from a browser:

ember -d --cookies-from-browser firefox "https://x.com/user/status/123"
  • Firefox is recommended — cookies read out of the box on any OS.
  • Non-Chrome Chromium browsers on Windows work too: Vivaldi, Opera.
  • Recent Chrome/Edge/Brave encrypt cookies (App-Bound Encryption) — reading them usually fails. Don't fight it, install Firefox.
Tip: a dedicated cookies browser. Install a browser you don't normally use (Firefox or Vivaldi), log into the services you plan to download from — and never open it again. The cookies stay valid, your main browser is untouched, and the session won't get logged out by accident.

Alternatives: cookies as a string --cookies "auth_token=...; ct0=..." or a file --cookies-file cookies.txt (Netscape format, same as yt-dlp). Pick a browser profile with --browser-profile.

Step 5. Python API

The basic loop — extract and download:

import ember

result = ember.extract("https://vimeo.com/76979871")
print(result.title, result.author)
for m in result.media:
    print(m.kind, m.quality, m.url)

paths = ember.download(result, "downloads/",
                       max_height=720, concurrency=6,
                       embed_metadata=True)

Useful checks:

ember.can_extract(url)                      # is the link supported
ember.available_qualities(result.media[0])  # e.g. [1080, 720, 480]
ember.ffmpeg_available()                    # is ffmpeg present
ember.probe_size(result.media[0])           # file size before downloading

Cookies in the API: ember.extract(url, cookies_from_browser="firefox") or a dict cookies={"auth_token": "..."}. Progress via the on_progress callback, playlists via extract_playlist(), author posts via extract_timeline(url, limit=30).

Typical embedding pattern — Ember first, yt-dlp as fallback:

if not ember.can_extract(url):
    return run_ytdlp(url)
try:
    result = ember.extract(url)
except ember.EmberError:
    return run_ytdlp(url)
ember.download(result, out_dir)

All errors inherit from ember.EmberError — catch that one. Details on every function and data model (Result, Media, Playlist) — in API.md.

Limitations

  • Instagram — anonymously often preview-only; cookies give full quality.
  • Reddit, Newgrounds, OK.ru — block datacenter/VPN IPs; fine on a home IP. --proxy helps.
  • Twitter/X — NSFW tweets require account cookies.
  • YouTube — out of scope, use yt-dlp for it.
